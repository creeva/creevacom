So I’ve talked about my hosting provider before, their tech support isn’t the brightest crayons in the box. It seems whatever issue I have with them, they don’t read what exactly I want. Key word analysis is great guys, I did it when I worked phone support at Symantec<\/a>. What I didn’t do is misinterpret everything the customer said on every single issue. If something isn’t working you have to fix it.<\/p>\n

Warning for the non-geeks that read my site, this is about to get technical.<\/p>\n

Let’s go back to the beginning and flash forward to present day. In September I really wanted to add SSL<\/a> for the administration section of my blog<\/a> – simple, easy, and secure. The biggest reason for this was the WordPress<\/a> 2.7 series offered better support for SSL, so now why the time to start instead of hacking around with plugins for support. This turned out to be not so simple.<\/p>\n

Online I found my hosting provider is supposed to offer a server self signed certificate, so I sent the following message to tech support:<\/p>\n

Is there a shared server certificate for SSL that I can use for my WordPress installation?<\/p><\/blockquote>\n

Now I’m fast and willing to pick through things on my own, so before technical support could reply I added this second message to my brand new ticket.<\/p>\n

I found the SSL manager in cpanel –<\/p>\n

1. Do you have instructions on properly generating your SSL key for use with
\nthis (wording might be wrong – but how to configure this so I can use it with
\nmy localhost\/wordpress domain?<\/p>\n

2. Do you instructions on using this with WordPress?<\/p><\/blockquote>\n

Simple and straight forward I thought. Here is what I got back:<\/p>\n

Hello,<\/p>\n

If you install the shared SSL for the domain, you will be getting warning when
\nyou take the site.<\/p>\n

It is better to buy SSL from third-party Godaddy.<\/p>\n

Please check the link given below for installing SSL.
\nhttp:\/\/www.cpanel.net\/support\/docs\/11\/cpanel\/sec_ssl.html<\/a><\/p>\n

In case you have any more queries, please don’t hesitate to contact us with
\nall the required details. We’ll be happy to assist you further.<\/p><\/blockquote>\n

Let’s throw the bit of broken english aside for a moment and break this down. I am aware (of course they don’t know this) that I will be getting a certificate mismatch error when I login to the SSL site. Problem one I see is that they are sending me a third party, yet they sell an SSL service – I find this odd, but Godaddy<\/a> (I’m assuming that was what he meant) is cheaper. Good customer service? Maybe. The other problem is that he sent me the instructions to doing this in Cpanel<\/a> 11, all fine and good, but they have all of their customer on Cpanel 10. The instructions were similar and I followed them. After implementing the changes I attempted to go to my website on HTTPS. Firefox <\/a>threw out the following message:<\/p>\n

Secure Connection Failed<\/p>\n

An error occurred during a connection to localhost\/wordpress.
\nSSL received a record that exceeded the maximum permissible length.
\n(Error code: ssl_error_rx_record_too_long)<\/p>\n

The page you are trying to view can not be shown because the authenticity of
\nthe received data could not be verified.<\/p>\n

* Please contact the web site owners to inform them of this problem.<\/p><\/blockquote>\n

I spent a day researching this message, BTW Firefox’s documentation of error message is terrible, hence why it took me a day instead of minutes. Also Firefox’s documentation is wrong as I would find out later. After a few days of trying to get this to work I gave up and wrote this off as a lost cause for the moment and focused on other things.<\/p>\n

Flash forward to yesterday, I decided to reopen this ticket since again I wanted SSL. I knew from experience that the shared certificate would not work properly, so I went and generated a free (valid) one from Instant SSL<\/a>. I wanted to do this before purchasing my own SSL certificate from Godaddy, just to make sure everything was working properly. I went through all the steps and I go the same error message. Well I saved the thirteen dollars a valid certificate would have cost, but wasn’t any closer to my end goal of an encrypted WordPress administration section. I still wanted this so I opened the original ticket back up (tech support hates that BTW) and added the following note:<\/p>\n

I’m back to this again – I was considering buying SSL instead of using the
\nshared cert – but having applied a cert from Comodo – I’m getting the same
\nerror in firefox:<\/p>\n

Secure Connection Failed<\/p>\n

An error occurred during a connection to localhost\/wordpress.
\nSSL received a record that exceeded the maximum permissible length.
\n(Error code: ssl_error_rx_record_too_long)<\/p>\n

The page you are trying to view can not be shown because the authenticity of
\nthe received data could not be verified.<\/p>\n

* Please contact the web site owners to inform them of this problem.<\/p>\n

If I try Internet Explorer it doesn’t even connect<\/p>\n

If I use digicerts SSL checker – http:\/\/www.digicert.com\/help\/<\/a> I get this:<\/p>\n

DNS resolves ‘localhost\/wordpress’ to 69.4.229.212
\nNo certificates were found.
\nOutput from ‘openssl s_client’ command:
\n13127:error:140770FC:SSL routines:SSL23_GET_SERVER_HELLO:unknown
\nprotocol:s23_clnt.c:585: CONNECTED(00000003<\/p>\n

Now following godaddy’s instructions it seems I need access to cpanel WHM –
\nwhich is something from a shared hosting perspective I need you to implement.
\nCan I get this done – also I’m perfectly fine using a self signed server
\ngenerated cert (which is what it currently is now)<\/p>\n

Since I’m not using a shopping cart and I’m only going to use SSL for
\nmaintenance of WordPress, it’s perfectly fine for me to receive the security
\nexception warning in Firefox since it will not be a customer facing error, I
\nonly want the encryption side, not the non-repudiation side of SSL.<\/p>\n

Also the instructions you sent me are for Cpanel 11 – currently my account uses
\ncpanel 10 when I login<\/p><\/blockquote>\n

For those technical people who understand what I’m trying to carry out, am I unclear on this? Please excuse my muddled use of non-repudiation. Also that stupid Firefox message I stated was vague, well it seems the server wasn’t even completing the connection properly – stupid Firefox error messages. This is the response I got back:<\/p>\n

For installing a dedicated SSL certificate on your domain, you will have to
\nbuy a static IP(dedicated IP) for your domain. For that you need to
\ncontact our billing department.
\nAfter obtaining a static IP, we will assign that IP to your domain. You can
\nupload your cert , key and CA bundle for the site to the server so that we
\ncan install the cert for you.
\nAlso please note that there will be some downtime for the site until the IP
\nchange propagates globally.<\/p>\n

Please confirm if you need a static IP for the domain, so that we can forward
\nthis to the concerned department.<\/p><\/blockquote>\n

Now what part my message did they not understand I did state that I was perfectly fine with an error message, that I wanted to use the shared certificate. I think I implied fairly well that I would prefer to use the shared certificate, that way I wouldn’t have to actually buy my own. This is the problem you get into when you do keyword scanning, at least the first guy with broken english understood my question better. My response:<\/p>\n

Like I said in my earlier e-mail – I do not need a static IP, nor do I need
\na dedicated SSL certificate – I’m fine using the standard shared SSL
\ncertificate – as was\/is advertised online that all plans came with a shared
\nserver SSL cert – that is what I would like to use and that is what is not
\nworking.<\/p>\n

Maybe they’ve finally understood my issue?<\/p>\n

We are checking your query regarding shared SSL in detail. We will get back to
\nyou with the updates soon.<\/p><\/blockquote>\n

Soon came about thirty minutes later:<\/p>\n

After careful review of this ticket, we have decided that you could be better
\nassisted in another department within our company.<\/p>\n

For this reason, we are transferring your ticket. Please expect a short period
\nof time where this ticket is not updated, as it is queued up at its
\ndestination.<\/p>\n

We make every effort possible to take care of every ticket as quickly as we
\ncan. We will contact you shortly.<\/p>\n

Thank you for understanding,<\/p><\/blockquote>\n

Was was nice enough to leave off the names of their technical support staff. What I don’t understand is why I have to go to another department – maybe I’m going to the server admins who actually make the changes? Any time I have an issue it seems that I have to go through a lot of back and forth in the technical support team. If you techies read my inquiries as unclear, please let me know. As I get more information I’ll be updating the comments.<\/p>\n

Grrrrrrrrrrrrrr<\/p>\n
